05. < title> The HTML Title element (< title>) defines the document's title that is shown in a browser's title bar or a page's tab. < title> This is a title tag< /title>
06. < body> < /body> The HTML < body> Element represents the content of an HTML document. There can be only one element in a document. < body> This is a body tag < /body>
07. < address> The HTML < address> element indicates that the enclosed HTML provides contact information for a person or people, or for an organization < address> This is a title tag< /address>
08. < article> The HTML < article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable < article> This is a body tag < /article>
09. < asside> The HTML < aside> element represents a portion of a document whose content is only indirectly related to the document's main content. < aside> This is a title tag< /aside>
10. < footer> The HTML < footer> element represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data or links to related documents < footer> This is a body tag < /footer>
11. < header> The HTML < header> element represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements. < header> This is a title tag< /header>
12. < hgroup> The HTML < hgroup> element represents a multi-level heading for a section of a document. It groups a set of < h1>–< h6> elements < hgroup> This is a body tag < /hgroup>
13. < main> The HTML < main> element represents the dominant content of the < body> of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application < main> This is a title tag< /main>
14. < nav> The HTML < nav> element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes < nav> This is a body tag < /nav>
15. < section > The HTML < section> element represents a standalone section — which doesn't have a more specific semantic element to represent it — contained within an HTML document. < section> This is a title tag< /secrion>
16. < blockquote> The HTML
Element (or HTML Block Quotation Element) indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation (see Notes for how to change it). A URL for the source of the quotation may be given using the cite attribute, while a text representation of the source can be given using the < cite> element
< blockqoute> This is a body tag < /blockqoute>
17. < dd> The HTML < dd> element provides the description, definition, or value for the preceding term (< dt>) in a description list (< dl>) < dd> This is a title tag< /dd>
18. < dir> The obsolete HTML Directory element (< dir>) is used as a container for a directory of files and/or folders, potentially with styles and icons applied by the user agent. < dir> This is a body tag < /dir>
19. < div> The HTML Content Division element (< div>) is the generic container for flow content. It has no effect on the content or layout until styled using CSS. < div> This is a title tag< /div>
20. < dl> The HTML < dl> element represents a description list. The element encloses a list of groups of terms (specified using the < dt> element) and descriptions (provided by < dd> elements). Common uses for this element are to implement a glossary or to display metadata (a list of key-value pairs). < dl> This is a body tag < /dl>
